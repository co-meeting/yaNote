# yaNote タッチ操作仕様・実装状況 (v1.4 本番)

## 基本操作対応表

以下の表は、マウス操作に対応するタッチ操作の仕様と、本番（v1.4）の実装状況を整理したものです。


| 操作内容     | マウス操作           | タッチ操作              | 現在の対応状況 | 補足                                    |
| -------- | --------------- | ------------------ | ------- | ------------------------------------- |
| パン操作     | 右クリック＋ドラッグ      | シングルタッチ＋ドラッグ       | ⭕ 実装済み  | キャンバス上でシングルタッチ＋ドラッグでキャンバス全体を移動        |
| ノード選択    | 左クリック           | タップ                | ⭕ 実装済み  | 選択時にハイライトを表示                          |
| ノード編集    | ダブルクリック         | ダブルタップ             | ⭕ 実装済み  | 編集モードに切り替え。編集中に空白タップで編集終了・選択解除        |
| ノード生成    | 空白部分をダブルクリック    | 空白部分をダブルタップ        | ⭕ 実装済み  | その位置に新規ノードを作成。10px未満の移動で離すと新規ノード      |
| ノード移動    | ノードをドラッグ        | 長押し(300ms)してからドラッグ | ⭕ 実装済み  | 長押し中は軽い拡大でフィードバック                     |
| 複数選択     | Shift+クリック      | 二本指タップ             | ❌ 未対応   | キーボード併用で Shift 操作は可能                  |
| 選択解除     | 空白部分をクリック       | 空白部分をタップ           | ⭕ 実装済み  | タップのみ（約10px未満の移動で判定）                  |
| ラバーバンド選択 | 空白部分で左クリック＋ドラッグ | タップ＋ドラッグ（空白）       | △ 部分対応  | 空白でドラッグ開始はパンになる。ラバーバンドはマウスまたはキーボード併用時 |

## 操作の判断基準と現状の仕様

1. **タイムアウトによる判定**:
  - タップ: 単発のタッチ（200ms以内、約10px未満の移動）
  - 長押し: 一定時間（300ms）同じ位置を押し続けてからドラッグでノード移動
  - ダブルタップ: 2回の連続タップ（間隔300ms以内）
  - **本番**: 長押し300ms・ダブルタップ300ms・タップ200msで実装済み
2. **ダブルタップ後のドラッグ距離で動作を決定**:
  - 10px未満 → ノードの編集／新規ノード作成
  - 10px以上 → 接続線の生成（ノード接続 or 自由接続線）
  - **本番**: 閾値10pxで競合せず実装済み
3. **パン操作とノード移動の区別**:
  - キャンバス空白をタッチしてドラッグ → パン移動
  - ノードを長押し(300ms)後にドラッグ → ノード移動
  - **本番**: 区別して実装済み

## 実装上の考慮事項と技術的メモ

1. **ダブルタップのズーム抑制**:
  - iOS/iPadOSではダブルタップによるズームを抑制する必要がある
  - **本番**: 実装済み（touchstart/touchend で preventDefault 等を適用）
2. **長押しの視覚的フィードバック**:
  - 長押し中は軽い拡大（scale(1.05)）でフィードバック
  - **本番**: 実装済み
3. **マルチタッチ**:
  - 複数指ジェスチャー（二本指タップ・ピンチ等）は未対応。シングルタッチ操作は実装済み
4. **タッチとマウス操作の優先順位**:
  - タッチデバイスでもマウスイベントが発火する環境では、タッチ用ロジックとマウス用ロジックの両方が存在。入力の判別は実装済み
5. **iOS/iPadOSのスクロール抑制**:
  - touch-action および canvas 上での preventDefault により抑制
  - **本番**: 対応済み
6. **PWA環境でのタッチ操作**:
  - PWAとして起動した場合も基本操作は同一仕様で動作
7. **初期表示サイズ**:
  - viewport の initial-scale 等でモバイル表示を調整済み
