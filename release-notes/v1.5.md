# yaNote リリースノート v1.5

**リリース日**: 2026-07-18

## 概要

v1.5 は、ソースファイルの分離とリリース作業のスクリプト化を中心とした、開発基盤の整備リリースです。機能・操作に変更はありません。

## 主な変更点

### ソースファイルの分離（内部リファクタリング）

- 1ファイルに収まっていた `index.html` から、CSS を `style.css`、JavaScript を `app.js` に分離
- ビルド不要・静的配信の構成はそのまま維持
- Service Worker のコアアセットに `style.css` / `app.js` を追加し、オフライン動作を維持

### リリース作業のスクリプト化

- `scripts/bump-version.mjs` — 6 箇所のバージョン表記を一括更新（期待件数と一致しなければエラー停止）
- `scripts/release.mjs` — バージョン更新＋リリースノート下書き生成＋README 変更履歴エントリ挿入
- `scripts/release-publish.mjs` — 検証 → commit / tag / push / GitHub Release 作成
- 手順書 `RELEASE.md` と Claude Code 用スキル（`.claude/skills/yaNote-release`）を追加
- `.gitignore` を追加（ローカル設定ファイルをコミット対象から除外）

### 開発ガイドラインの刷新

- `docs/yaNote_DevGuide.md` をコーディングエージェント時代に合わせて更新
- リポジトリに `CLAUDE.md`（Claude Code 向けプロジェクトガイド）を追加

## 互換性

- 破壊的変更なし
- 既存のノートデータ（JSON・localStorage・共有URL）との互換性を維持
- Service Worker のキャッシュ名を v1.5 に更新（PWA は自動的に新バージョンへ更新されます）

## ドキュメント更新

- `README.md` — 変更履歴の追加、リリース手順への参照を追加
- `RELEASE.md` — リリース運用ガイドを新規追加
- `CLAUDE.md` / `docs/yaNote_DevGuide.md` — 開発ガイドラインを更新
